Как разработчик уровня senior, cоздай полноценный Python-проект под названием "AI Client PDF Generator".

Задача проекта:
Автоматически формировать PDF-отчёты по диалогам с клиентами, используя ИИ и HTML-шаблоны.

Технические требования:

1. Используемые библиотеки:
- openai (или anthropic, если подключаем Claude)
- jinja2 — для HTML-шаблонов
- weasyprint — для генерации PDF из HTML
- flask — для API-запросов (опционально)
- python-dotenv — для хранения API ключа в .env

2. Архитектура проекта:
project_root/
├── main.py # основной файл
├── templates/
│ ├── report_template.html # HTML-шаблон отчёта
├── reports/
│ └── (сюда сохраняются готовые PDF)
├── utils/
│ ├── ai_processor.py # взаимодействие с моделью (OpenAI/Claude)
│ ├── pdf_generator.py # функции HTML→PDF
├── .env # API ключ
├── requirements.txt # зависимости

3. Логика работы программы:

3.1. Initial setup
Проект должен подключать openai или любой совместимый LLM (gpt-4, gpt-3.5, llama 3 и т.д.), который получен через AI-провайдера, так же как это сделано в текущих скриптах ai_direct.py, zai_direct.py, proxy_api.py, gen_api.py и openai_client.py с теми же api-ключами как и в файле .env. Не удаляй ключи из текущего файла .env. OPENAI_API_KEY нужен не отдельно, а от того провайдера, которого выбрал пользователь.
3.2 Settings
При первом запуске, пользователь выбирает ai-провайдера, модель, температуру, токены из доступных вариантов в скриптах ai_direct.py и openai_client.py. Пользоватль должен иметь возможность смениь настройки.

3.3 Пользователь отправляет транскрибацию диалога (текстовый файл или ввод вручную).

3.4 Код вызывает функцию process_dialog_with_ai(text) из ai_processor.py, которая:
- Передаёт транскрибацию в OpenAI или Claude.
- Возвращает структурированные данные в JSON:
{
"client_name": "...",
"topic": "...",
"main_request": "...",
"mood": "...",
"next_steps": "..."
}

3.5 Далее pdf_generator.py:
- Подставляет эти данные в HTML-шаблон через Jinja2.
- Конвертирует HTML в PDF.
- Сохраняет PDF в папку `/reports` с именем вроде `report_2025-11-13_15-30.pdf`

3.6 В main.py всё объединяется:
- Считываем транскрибацию.
- Отправляем в ИИ.
- Генерируем PDF.
- Печатаем “Отчёт успешно создан: reports/report_….pdf”.
